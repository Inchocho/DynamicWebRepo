HTTP 응답 상태 코드

상태코드		상태설명
200			요청이 성공적으로 처리 (200번대의 경우 성공적으로 처리된 코드)
301			요청한 자원이 이동함
304			클라이언트가 임시 보관한 응답결과와 다르지 않다
400			잘못된 요청이다	
404			요청한 자원을 못 찾았다 ##404 error page not found##
500			서버 내부에서 오류가 발생했다 

등등 


src
자바 소스 파일을 두는 폴더
이 폴더에 서블릿 클래스나 필터, 리스너 등
필요한 모든 자바 클래스 파일을 두게 된다
프로퍼티 파일도 이 폴더에서 관리한다

build / classes
컴파일된 자바 클래스 파일(.class)들이 놓이는 폴더이다
패키지에 소속된 클래스인 경우 이 폴더에 해당 패키지가 자동으로 만들어진다

WebContent(webapp) || WebContent가 webapp으로 대체되었음(원래는 스프링에서는 webapp)
.html, .css, .js, jsp, 이미지 파일등
웹 컨텐츠를 두는 폴더이다(프론트단 파일)
웹 어플리케이션을 서버에 배치할 때 이 폴더의 내용물이 그대로 복사된다

WebContent(webapp)/WEB-INF
웹 어플리케이션의 설정과 관련된 파일을 두는 폴더이다
이 폴더에 있는 파일은 클라이언트에서 요청 할 수 없다
따라서 html이나 javascript등 클라이언트에서 요청 할 수 있는
파일을 이 폴더에 두어서는 안 된다

class의 모임 -> 패키지 
패키지의 모임 -> 라이브러리
라이브러리의 모임 -> 모듈(컴포넌트)
모듈(컴포넌트)의 모임 -> 컨테이너

컨테이너(큰단위) > 모듈 > 라이브러리 > 패키지 > 클래스(작은단위) 

WebContent(webapp)/WEB-INF/web.xml
웹 어플리케이션 배치(batch) 설명서(Deployment Descriptor) 파일이다
DD파일이라고 부른다. 서블릿이나 필터, 매개변수, 기본 웹 페이지등
웹 어플리케이션 컴포넌트들의 배치 정보를 이 파일에 작성한다
서블릿 컨테이너는 클라이언트의 요청을 처리할 때 이 파일의 정보를 참고하여
서블릿 클래스를 찾거나 필터를 실행하는 등의 작업을 수행한다

외부 API기능 사용시 lib에 작성하게 될것이다

WebContent(wepapp)/WEB-INF/lib
자바 아카이브(Archive) 파일(.jar)을 두는 폴더이다
아카이브란? 우리말로 기록 보관소란 뜻이다
앞으로 필요로 하는 다양한 기능들을 외부에서 가져와 사용할 때
이곳에 파일을 둔다.



xml 파일이란?

Extensible Markup Language 파일이다
일반 텍스트 파일과 다를 건 없다
단지 구조화 되어서 내용을 전달하도록 하여
다른 이들이 쉽게 구분하여 내용을 볼 수 있게 만들어져 있어
내용들을 파악하는데 유용하다
